**Write implementation notes to the task**

Write notes before the commit for a given task using the configured task-system interface. If the project-local deployment workflow requires a pre-deploy commit, write notes before that commit.

Keep implementation notes concise and evidence-focused. For low-complexity tasks, list only touched files, non-obvious decisions, and exact verification outcomes. Summarize failures and blockers in one or two sentences instead of pasting long command output unless the exact output is needed to diagnose an unresolved blocker.

Include every file that was created or modified. Be specific — path and one-line reason.
